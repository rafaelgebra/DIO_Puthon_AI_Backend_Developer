VCS é o sistema de controle de versão. Ele é usado para ajudar e dar segurança ao trobalho porque muitas pessoas podem trabalhar no mesmo código com segurança. Os tipos de VCS são: (CVCS) VCS centralizado -> ex: CVC, Subversion Nesse caso só tem um servidor de faz o controle de versão, então caso o servidor fique fora do ar, ninguem consegue trabalhar. porque depende do servidor VCS distribuido (DCVS) Git, MErcurial Nesse caso, o computador tem uma copia do banco de versão (duplicado) para cara computador conectado. Então caso o servidor caia ou fique sem internet ainda pode trabalhar porque uma copia localmente no computador. git commit -> gravar alterações no repositório. git pull -> "puxa" as alterações do repositório remoto para o local (busca e mescla as alterações). git puch -> "empurra" as alterações do reposito local para o repositório remoto. 1° Configurar nome e e-mail no Git. nome = git config --global user.name "Rafael_teste" e-mail = git config --global user.email "rafaelrodriguesgebra@gmail.com" 2° Configurar o
